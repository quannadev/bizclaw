# Self-Evolution System

BizClaw's Self-Evolution system allows agents to learn and improve from usage patterns, adapting their behavior over time while maintaining safety guardrails.

## Overview

The Self-Evolution system consists of three main components:

```
┌─────────────────────────────────────────────────────────────────┐
│                     Self-Evolution System                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐      │
│  │   Metrics    │───►│  Analysis    │───►│  Adaptation  │      │
│  │  Collection  │    │   Engine     │    │   Engine    │      │
│  └──────────────┘    └──────────────┘    └──────────────┘      │
│         │                  │                   │                  │
│         ▼                  ▼                   ▼                  │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐      │
│  │  History    │    │ Suggestions  │    │  Guardrails  │      │
│  │  Storage    │    │  & Analysis  │    │  & Safety    │      │
│  └──────────────┘    └──────────────┘    └──────────────┘      │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## Features

### 1. Metrics Collection

The system continuously collects metrics about agent behavior:

- **Conversation metrics**: Message counts, context switches
- **Tool usage**: Which tools are used, success rates
- **Response patterns**: Response time, length, format
- **User satisfaction**: Ratings and feedback (when available)
- **Error tracking**: Error counts and types

```rust
// Example: Collecting metrics
let metrics = AgentMetrics {
    agent_id: "my-agent".to_string(),
    timestamp: Utc::now(),
    conversation_count: 10,
    message_count: 100,
    tool_usage: HashMap::from([("browser".to_string(), 5)]),
    tool_success_rate: HashMap::from([("browser".to_string(), 0.95)]),
    avg_response_time_ms: 1500,
    user_satisfaction: Some(4.5),
    error_count: 2,
    context_switches: 3,
};

evolution.collect_metrics(metrics);
```

### 2. Suggestion Analysis

The analysis engine identifies patterns and suggests improvements:

#### Communication Style Analysis
- Response length optimization
- Tone and formality adjustments
- Code explanation levels

#### Tool Usage Analysis
- Underutilized tools
- Inefficient tool chains
- Success rate patterns

#### Domain Expertise
- Error rate patterns
- Knowledge gaps
- Skill improvements

```rust
// Generate suggestions based on collected metrics
let suggestions = evolution.analyze_and_suggest("my-agent");

for suggestion in suggestions {
    println!("{:?}", suggestion);
}
```

### 3. Auto-Adaptation with Guardrails

The adaptation system applies improvements with safety guardrails:

#### Guardrail Levels

| Level | Description |
|-------|-------------|
| **Critical** | Requires manual approval |
| **High** | Limited changes per day |
| **Medium** | Standard adaptation |
| **Low** | Frequent updates allowed |

#### Configuration

```rust
let config = EvolutionConfig {
    enabled: true,
    collection_interval_secs: 300,
    analysis_interval_secs: 3600,
    adaptation_threshold: 0.8,
    guardrails_enabled: true,
    max_changes_per_day: 10,
};
```

## Usage

### Basic Setup

```rust
use bizclaw_agent::self_evolution::{SelfEvolution, EvolutionConfig};

let config = EvolutionConfig::default();
let evolution = SelfEvolution::new(config);
```

### Collecting Metrics

```rust
// After each agent interaction
evolution.collect_metrics(AgentMetrics {
    agent_id: agent_id.clone(),
    timestamp: Utc::now(),
    // ... other metrics
});
```

### Periodic Analysis

```rust
// Run every hour (or based on your analysis interval)
let suggestions = evolution.analyze_and_suggest(&agent_id);

for suggestion in suggestions {
    if suggestion.confidence >= 0.8 {
        let change = evolution.apply_adaptation(&suggestion)?;
    }
}
```

### Rollback

```rust
// If a change causes issues
evolution.rollback_adaptation(&change_id)?;
```

## Configuration Options

### EvolutionConfig

| Option | Type | Default | Description |
|---------|------|---------|-------------|
| `enabled` | bool | true | Enable/disable self-evolution |
| `collection_interval_secs` | u64 | 300 | How often to collect metrics |
| `analysis_interval_secs` | u64 | 3600 | How often to run analysis |
| `adaptation_threshold` | f32 | 0.8 | Minimum confidence to apply changes |
| `guardrails_enabled` | bool | true | Enable safety guardrails |
| `max_changes_per_day` | usize | 10 | Maximum changes per agent per day |

## Safety Features

### Guardrails

1. **Critical Changes Blocked**: Major behavior changes require manual approval
2. **Daily Limits**: Prevents rapid accumulation of changes
3. **Rollback Support**: Easy rollback of problematic changes
4. **Audit Trail**: Full history of all changes

### Monitoring

```rust
// Get all suggestions with status
let pending = evolution.get_suggestions(&agent_id, Some(SuggestionStatus::Pending));
let applied = evolution.get_suggestions(&agent_id, Some(SuggestionStatus::Applied));
```

## Best Practices

1. **Start with guardrails enabled**: Review changes before auto-applying
2. **Monitor closely initially**: Watch for unintended behavior changes
3. **Use rollback liberally**: Don't hesitate to undo problematic changes
4. **Review capabilities files**: Ensure auto-generated content is accurate
5. **Gradually increase limits**: Start conservative, expand as confidence grows

## API Reference

### Types

- `EvolutionConfig` - Configuration for the evolution system
- `AgentMetrics` - Metrics collected from agent interactions
- `EvolutionSuggestion` - A suggested improvement
- `AdaptationChange` - A change that was applied
- `Capabilities` - Auto-generated agent capabilities

### Methods

- `SelfEvolution::new(config)` - Create a new instance
- `collect_metrics(metrics)` - Record agent metrics
- `analyze_and_suggest(agent_id)` - Generate suggestions
- `apply_adaptation(suggestion)` - Apply a suggestion
- `rollback_adaptation(change_id)` - Rollback a change
- `generate_capabilities_md(agent_id)` - Generate capabilities file
- `get_suggestions(agent_id, status)` - Get suggestions by status

## Troubleshooting

### Suggestions not appearing

1. Ensure metrics are being collected regularly
2. Check that enough metrics history exists (minimum ~10 data points)
3. Verify the agent ID matches exactly

### Changes not applying

1. Check if guardrails are blocking changes
2. Verify daily limits haven't been exceeded
3. Ensure confidence threshold is met

### Poor suggestions

1. Collect more diverse metrics
2. Increase analysis interval for better patterns
3. Review metrics for data quality
